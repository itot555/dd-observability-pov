dd-observability-pov — single-host Architecture

Java + Python を 1 台の EC2（Amazon Linux 2023）に集約したコスト最適化構成。Datadog SSI（Single Step Instrumentation）の動作確認にご利用いただけます。
📦 Region: ap-northeast-1 🏷️ Prefix: pov (variable: name_prefix) 🖥️ OS: Amazon Linux 2023 💰 NAT Gateway: なし（コスト削減）

Architecture Diagram

Client → Java → Python → PostgreSQL の 4 階層トレースを 1 台の EC2 上で再現。 Datadog Agent は SSI でインストールするためアプリ側のコード変更は一切不要。

Endpoints

Java Spring Boot (8080) で公開。Java → Python (localhost:8000) → 必要に応じて Postgres (5432) と段階的に呼び出される。

Basic endpoints (Java → Python)

Public PathPython PathBehaviorResult
GET /helloGET /api/data シンプルな JSON エコー 200 OK
GET /timeoutGET /api/timeout 30% の確率で 10 秒スリープ → Java 側 5s timeout で 504 504 timeout 30%
GET /errorGET /api/error 20% の確率で 500 を返す 500 error 20%

Database endpoints (Java → Python → Postgres)

Public PathPython PathBehaviorDB Spans
GET /db/normalGET /api/db/normal users JOIN orders を 1 クエリで LIMIT 200 取得 1 span
GET /db/n1GET /api/db/n1 users 20 件取得 → 各 user の orders をループで個別取得（N+1） 21 spans
GET /db/long-runGET /api/db/long-run SELECT pg_sleep(35), count(*) FROM users 35s span

Datadog Integration (SSI)

Before / After POV ストーリー

Phase状態操作
Before Java + Python が同一ホストで連携動作。Datadog Agent 未インストール。可観測性ゼロ。 —
After SSI で Agent インストール後、アプリ再起動するだけで APM・ログ・メトリクスを取得 DD_APM_INSTRUMENTATION_LIBRARIES="java:1,python:3" を付けて 1 コマンドでインストール → アプリ再起動

terraform output demo_step1_before 〜 demo_step4_after で各ステップのコピペ用コマンドを出力します。

期待されるトレース（After フェーズ）

  • Service Map: pov-java-front → pov-py-back → postgres の 3 階層（同一ホストでも別サービスとして認識）
  • /db/normal: 1 トレース内に postgres.query span が 1 個
  • /db/n1: 1 トレース内に postgres.query span が 21 個（N+1 アンチパターン）
  • /db/long-run: postgres.query span が 35 秒幅で表示
  • Logs: dd.trace_id が埋まり、トレース ↔ ログ相関が動く

Operations

Apply / Destroy

cd terraform/single-host
terraform init
terraform plan
terraform apply     # RDS 作成に 10〜15 分
terraform destroy   # 残骸なしで完全削除

Destroy 容易化: RDS は skip_final_snapshot / deletion_protection=false / backup_retention_period=0 / delete_automated_backups=true / Performance Insights・Enhanced Monitoring・CloudWatch Logs export を全無効化。

EC2 アクセス（直接 SSH）

ssh -i terraform/single-host/keys/pov-ssh-key ec2-user@<ec2_public_ip>
